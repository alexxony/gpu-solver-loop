GPU-Solver

측정 피드백으로 분류 규칙 자체가 진화하는 GPU 커널 최적화 루프
A GPU-kernel optimization loop whose classification rules evolve from measurement feedback

루프 한 바퀴 = generate → gate → profile → hypothesize → measure → evolve. 차별점은 마지막 단계: 측정이 룰을 되먹여 오탐 룰을 폐기(retire)하고 맞는 룰의 신뢰도를 올린다. CUDAMaster 등 선행 시스템은 이 되먹임이 없다(룰표 정적). rule DB evolution = 유일 차별점

① 성능 gain — matmul 6.4×

측정→가설→재작성→더 빠른 커널 / measure → hypothesize → rewrite → faster kernel (A100)

fp32 ampere_sgemm 9.6 ms → TF32 cutlass_tensorop 1.5 ms. 루프가 fp32_no_tensorcore 룰을 발화 → TF32 코드 선택 → 6.4× 실측. 커널명이 달라짐(sgemm ≠ tensorop) = 플래그 조작이 아니라 진짜 다른 커널 선택.
The loop selected a genuinely different kernel — not a flag tweak. Single compute-bound example (honest scope).

② 룰 진화 — 측정으로 오탐 룰 폐기

rules retire on measurement — what prior systems can't do (T4, 8 rounds)

T4에서 reg_pressure 오탐이 8라운드 발화. 진화 ON: 4라운드째 측정 누적으로 retire=1 → 다음 적격 룰로 전환. 진화 OFF: retire=0, 영원히 오탐 반복.
⚠ latency는 두 트랙 모두 평평(48.4 ms, T4 fp32 천장) = gain 아님. 차이는 오직 retire에서만 — 이게 선행이 못 하는 것.
The difference is the retire, not speed.

세 축 각각 실증 / three axes, each demonstrated

  • gain 축 — matmul 6.4× (A100, 위 ①)
  • 진화 축 — reg_pressure retire (T4, 위 ②) + sigmoid 오탐 retire (A100)
  • 칩 축 — 같은 신호, 칩 따라 다른 룰: A100=fp32_no_tensorcore 발화 vs T4=가드 차단 (chip guard, 아키텍처 다이어그램 참조)

미입증(future work): 다문제 gain 일반화, 한 문제서 두 축 동시(구조적 난제로 확인).

데이터 = 실측 하드값 (A100 matmul 라운드, T4 8-round gain_compare 로그). 정직 유지: null 결과는 물리 천장으로 명시.
